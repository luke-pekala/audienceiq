About AudienceIQ

v1.0-beta · Built by Luke

What it is

AudienceIQ is an audience analysis tool for technical writers, documentation leads, and anyone responsible for producing written content that needs to land with a specific reader.

Most documentation fails because writers write for themselves — organised around their own knowledge, calibrated to their own vocabulary, paced to their own patience. AudienceIQ helps you escape that trap before you write a word.

How it works

Describe your audience in plain language. AudienceIQ analyses the description and returns:

  • A knowledge profile with scored dimensions (technical depth, domain expertise, reading patience, jargon tolerance)
  • Knowledge gaps — the specific things your audience won't know that you'll assume they do
  • Tone and vocabulary guidance with explicit do/avoid lists
  • A reader journey map showing how your reader moves through a document
  • An exportable audience brief you can share with your team

Who built it

AudienceIQ is App B in a 26-app documentation tooling portfolio, built as part of a structured journey toward a Product/Tools Developer role. The portfolio covers the full stack from vanilla JS through React, Node.js, Supabase, and Claude API integrations.

The app ideas are grounded in a systematic analysis of technical writing literature — 347+ validated pain points extracted from 11 books, ranked by cross-book confirmation count. Audience mismatch is one of the highest-confirmed pain points across the entire corpus.

What's next

The current version uses heuristic analysis. The next version will integrate the Claude API for deeper, more contextual persona generation. The upgrade slot is already built into the codebase — see src/js/engine.js.
